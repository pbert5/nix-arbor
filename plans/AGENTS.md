# Plan Instructions

Apply these rules to planning documents under `plans/`.

- Plans may describe intended designs, experiments, and migration paths.
- Clearly distinguish planned work from behavior that is already implemented.
- When implementation happens, update durable docs or path-local instructions
  so current truth does not live only in a plan.
- Keep plans scoped to their topic. Do not use plan files as replacement
  `AGENTS.md` instructions for active code paths.
